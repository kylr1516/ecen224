I, Dr. Lundrigan, am piloting a new class sophomore-level class, ECEn 224, and I need your help! I need motivated students interested in shaping the sophomore student experience for years to come. The class will be taught full-time starting next year (Fall 2023). In the meantime, I would like to get a feel for how students enjoy the content, labs, class pacing, etc. I am looking for about 20 interested sophomore students to help pilot this class. If you are interested, contact me at lundrigan@byu.edu.

The class will cover the following topics:
  • C Programming. After taking this class, you will be an advanced C programmer.
  • Machine Representation. You will learn how a computer translates a C program into a machine code
  • Computer Organization. Learn how computers are structured to be efficient and flexible.
  • Complex System. The semester project is to build a smart doorbell system. You will gain experience with connecting multiple systems to build something really cool.

This class will also have a practical skills component where we will teach you how to use Linux command line and tools like git, gdb, make, etc.

Frequently Asked Questions

What credit do I get for taking this class?

If you are a Computer Engineering student, you will get credit for ECEn 424. You still must take ECEn 220.

If you are an Electrical Engineering student, this class will replace ECEn 220. If you have already taken ECEn 220, then this class will count as credit for ECEn 323.

If you are an IT&C student, you get credit for IT&C 252.

Why would I mess up my schedule to take this class?

Here are a few reasons to go through the trouble of moving your schedule around so you can take this class. First, you get to do something that not very many students have the opportunity to do—directly impacting a class's content for years to come. Since this is a pilot, I will take all student feedback very seriously. You will be a partner with me in building this class together.

Second, you will have an advantage over your peers. This content is so crucial that we are moving it to the sophomore year, starting next year (as opposed to an optional senior-level class). You will have the opportunity to learn material your peers won't learn. As a result, you will be more competitive for internships and other opportunities.

Lastly, as part of this class, a practical skills component will teach you essential skills like Linux command line, git, gdb, etc. This material is not taught anywhere else in our program. You get credit for learning these skills, and your peers have to learn them on their own.

Why is this class 4 credits?

Since this course has a three-hour lab, plus a regular lecture schedule, we felt that 4 credits would be a better representation of how much work a student will do for this class.

What is the time slot on Tuesday?

The Tuesday timeslot is an optional recitation session where you will meet with a TA to discuss the course content, practice example problems, and learn important skills that will help you in the labs. While it is not required, I highly recommend it.

What are the prerequisites for this class?

CS 111 (or CS 142)

What will I learn about in this class?

Here are the learning objectives
  • Binary Representation and Computation. Ability to represent numbers in binary and perform binary operations.
  • C Programming. Ability to program using the C programming language.
  • Machine Representation of Programs. Ability to represent programs in assembly language, translate between C and assembly language, and evaluate security vulnerabilities.
  • Computer Organization. Ability to identify basic computer components, including memory hierarchy, I/O, and central processing unit (CPU), and explain how these components work together to create a computer system.
  • I/O Programming. Ability to build a system that securely interacts with I/O devices and computer networks through C programming.

Is there a book?
